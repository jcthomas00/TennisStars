Typescript Functions

Basic JS Function Declaration

A function is a set of statements to perform a specific task. Functions organize the program into logical blocks of code.


    function  function_name( ) { } // function definition
    function_name() // function invocation

Positional parameters in Typescript Function

  • Parameters can have a specified type by using the syntax:
    • param_name: type
  • Parameters without a specified type are considered of type any
  • Parameters can be made optional with '?' by using the syntax
    • param_name?: type
  • Parameters can be given a default value with '=' just like in regular JS functions.
  • Note that a parameter can't be optional and have a default value

    function  function_name(p1: string = "hello", p2?: number) {   
        console.log(p1);                                
        console.log(p2);                                
    }

Typescript Function Types

  • Function types can be written as a call signature in an object type:
    • The following code is defining a function type with one parameter, named a (of type string) that doesn’t return anything:
    • type GreetFunction = {(a: string): void;};
  • The below code is exactly the same code as above but written in shorthand form (similar to an arrow function):
    • (a: string) => void
  • A type alias can be used to name a shorthand function type:
    • type GreetFunction = (a: string) => void; 

    // define function type  
    type GreetFunction = (a: string) => void;     

    // implement function of type GreetFuncti
    function printToConsole(s: string) {    
        console.log(s);                     
    }                                           

    // implement function that gets a parameter of type GreetFunction
    function greeter(fn: GreetFunction) {   
        fn("Hello, World");                 
    }                                           
    
    // invoke function greeter               
    greeter(printToConsole);

Typescript Function Interfaces

  • Function interfaces are used exactly the same way as types but have a slightly different syntax:
    • The following code shows the same call signature as the type examples above, but as an interface:
    • interface SearchFunc {(a: string): void;}

Overloading Functions in Typescript

What Is Function Overloading?

Overloaded functions have the capability to operate differently on the basis of the input provided to them. In other words, a program can have multiple methods with the same name with different implementation.

Overloading in Typescript

  • A function with multiple call signatures is defined as an overloaded function
  • Steps to overload a function:
    1. Declare multiple functions with the same name but different function signature. The signature can differ in:
      • The data type of the parameter
      • The number of parameters
      • The sequence of parameters
    2. The declaration must be followed by the function definition and it must account for all of the signatures.
    3. Lastly, call the overloaded function

    //STEP 1
        // Example of function declarations with different data types
        function fn1(string):void; 
        function fn1(number):void; 

        // Example of function declarations with different sequence of parameters
        function fn2(n1:number,s1:string):void; 
        function fn2(s:string,n:number):void;

        // Example of function declarations with a different number of parameters
        function fn3(n1:string):void; 
        function fn3(x:number,y:string):void;

    //STEP 2 

        // Implementing the function (only for fn3 declarations)
        function fn3(x:string|number,y?:string):void { 
            console.log(x); 
            console.log(y); 
        } 
    
    //STEP 3

        // Calling overloaded functions
        fn3("abc") 
        fn3(1,"xyz");

Overloading Function Expressions

Since we can use type declaration to define function's call signature, we can also use it to overload the function by providing multiple signatures in side the definition. 

    // Overloaded function expression
    type Reserve = {
        (from: Date, to: Date, destination: string): void
        (from: Date, destination: string): void
      }
      
      // Function implementation
      let reserve: Reserve = (
        from: Date,
        toOrDestination: Date | string,
        destination?: string
      ) => {
        if(typeof toOrDestination === "string"){
            console.log(`Have a great trip to ${toOrDestination} on ${from.toLocaleDateString()}`)
        }else{
            console.log(`Have fun in ${destination} from ${from.toLocaleDateString()} to ${toOrDestination.toLocaleDateString()}`)
        }
      }

Function Overloading Tips

  • The signature of the implementation is not visible from the outside. When writing an overloaded function, you should always have two or more signatures above the implementation of the function.
  • Always prefer parameters with union types instead of overloads when possible
  • When using overloads, try to keep your implementation’s signature as specific as possible to make it easier to implement the function (i.e. don't use any )

Typescript Modules

Modules in Javascript

We have been using Javascripts ES Modules format to handle modules using the import/export syntax

  • In TypeScript any file containing a top-level import or export is considered a module
  • You can make TypeScript treat a file as a module by adding export {}; to it
  • TypeScript adds the ability to import and export types on top of ES Modules functionality

    // @filename: animal.ts
    export type Cat = { breed: string; yearOfBirth: number };
    export type Dog = { breeds: string[]; yearOfBirth: number };
    export type Bird = { breed: string; yearOfBirth: number; color: string };

    // @filename: main.ts
    import type { Cat, Dog } from "./animal.js";    //mass import with import type
    import { type Bird } from "./animal.js";        //inline type import
